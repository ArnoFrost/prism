# Prism — 对内沟通口径 `v2.0.0-beta.1`

> 面向 leader / 团队沟通的简要说辞（≤300 字）

---

## 一句话版

Prism 让 AI 协作从一次性对话变成可接续、可复盘的过程，不改主仓，不打乱现有节奏；v2.0 把治理能力降级为可选项，v2.1 补齐了对外文档矩阵和发布门禁——开箱用得轻、对外协作有规可循、破坏性变化有机器护栏。

## 简版口径

**问题**：当前 AI 辅助开发存在上下文断裂、每次重新解释、过程不可追踪的问题，导致返工和知识丢失。

**Prism 做什么**：在现有项目里加一层轻量桥接——通过软链接把 AI 协作的上下文、评审记录、任务状态挂到项目旁边，不侵入主仓，不改变现有流程。最小依赖只有 `SDK + Vault + uv`。

**v2.0 的关键变化**：默认体验更轻——workflow / 痕迹义务家族明确为可选项，不接入治理框架也能用；用户首屏不再被内部评审/决策链路污染；deprecated 命令一次性物理移除，不再背双 alias 包袱。

**v2.1 的增量**：补齐 5 件套——v1.x→v2.0 的迁移指引、贡献者入口与受众分层、默认面 WARN 扫描器、breaking 发布门禁（CI 强制要求 CHANGELOG + 迁移文档同步）。对外文档第一次有了统一入口和机器护栏。

**当前状态**：v2.0.0-beta.1 内部观察期；架构层面没有计划中的破坏性变更，重点在 dogfood 验证与稳定性确认。

**下一步**：选一个小项目做试点，验证"AI 协作可追踪、可接续"对团队是否有实际价值。

**不做什么**：不替代现有项目管理流程，不要求全员使用，不接管代码仓库。先当样板试，跑通再说。

## 常见追问准备

| 追问 | 回答要点 |
|------|---------|
| 和现有工具什么关系 | 不替代，只在 AI 协作这个维度补一层可追踪的过程 |
| 要多大投入 | 零侵入接入，一条命令初始化，试点成本极低；v2.0 默认 lenient，不需要先学治理框架 |
| 能量化收益吗 | 减少重复解释的时间、评审产物可复用、上下文不断裂——可以做 before/after 对比 |
| 只能 Mac 用？ | 当前版本基于 macOS 软链接，覆盖 Mac 开发场景。跨平台是后续议题 |
| 和 XX 平台的区别 | Prism 是本地工具不是平台，不需要部署服务，不收集数据 |
| 治理框架强不强制 | 不强制。core contract 仅 `SDK + Vault + uv`，多角色评审 / 痕迹义务等是 v2.0 起明确的可选增强 |
