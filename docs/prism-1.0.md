# Prism 1.0 — 当前定位与成熟度说明

> 这不是安装文档，也不是完整架构文档。它只回答四个问题：**Prism 现在是什么、为什么算成立、边界在哪里、接下来还差什么。**

---

## 一句话版本

Prism 1.0 已经不是“几个 AI skill + 一堆脚本”的松散集合，而是一个**本地优先、分层清晰、带 CLI 自省能力的个人 AI 工作流管线**。

---

## 它现在是什么

Prism 当前由四个正交载体共同组成：

| 层 | 角色 | 当前状态 |
|----|------|---------|
| **SDK** | 协议、模板、CLI、内置 workflow 主干 | 已是当前最成熟、最稳定的主干层 |
| **Skills** | 可分享的通用能力层 | 保持轻量，不再承担系统主干职责 |
| **Env** | 设备/终端/本地习惯层 | 设计已收敛，与 SDK 解耦协作 |
| **Workspace** | topic / review / scope / plan 等状态容器 | 已成为项目级协作状态的固定落点 |

因此，Prism 更接近一个**个人 AI 工作操作系统的早期稳定版**，而不只是“提示词仓库”或“技能市场”。

---

## 为什么说它已经成立

### 1. 主干已经收敛到统一 CLI 面

023 / 024 之后，Prism 不再依赖分散的 skill 入口来承担主流程，而是把高频 workflow 动作上收到了 `prism <verb>`：

- `prism finalize`
- `prism tidy`
- `prism status`
- `prism digest`
- `prism --json manifest`

这意味着 Prism 已经有了一个明确的、可自描述的命令面，而不是只靠“记住有哪些 skill”来使用。

### 2. 它已经具备系统自省能力

Prism 现在不仅能做事，还能描述自己、检查自己、治理自己：

- `prism --json manifest`：导出当前 verb registry
- `prism sync`：观察 SDK / Skills / Env 三仓状态
- `bin/doctor`：体检、回滚、输出 release health JSON

当一个系统开始具备这些能力时，它就从“脚本集合”跨到了“可治理系统”。

### 3. 分层边界已经基本自洽

当前边界已经比较清楚：

- **SDK** 承担主流程与契约
- **Skills** 承担轻量可分享能力
- **Env** 承担设备与终端环境
- **Workspace** 承担状态

这让 Prism 可以同时保持：

- 本地优先
- 零侵入接入
- 多仓独立版控
- 软链接桥接

这些特征组合起来，构成了它独特的“极客味”。

---

## 它还不等于什么

Prism 1.0 已经成立，但它**还不等于**：

- 一个已经完成大规模泛化验证的通用产品
- 一个对多人团队协作已完全验证的控制面
- 一个已经把所有文档、入口、叙事全部打磨完成的公开发行版

换句话说，当前主要缺口不是“主干架构未成立”，而是**泛化验证与叙事收尾**。

---

## 当前边界在哪里

### 已经成立的部分

- SDK 主干、内置 workflow、CLI contract 已成型
- Env / SDK / Skills / Workspace 四层关系已基本自洽
- Workspace topic 机制已能支撑持续评审、决策、scope 派生与状态追踪
- Prism 自己以及至少一个非 Prism 项目已经验证过这套 workflow

### 仍然保留的边界

- 第二/第三个非 Prism 项目的长期持续验证仍不够
- 新用户从零接入的 smoke test 还值得再跑一轮
- `Quick Topic` 等降低门槛的轻量入口仍可补
- 文档叙事直到 025 专项完成前，都仍处于“实现追平文档”的过程里

---

## 现在最适合做什么

Prism 当前最不需要的是再发起一轮大重构；最适合的是：

1. **继续真实使用一段时间**  
   观察 `finalize / tidy / status / digest` 是否真的顺手。

2. **补齐文档叙事**  
   让 README / architecture / 工具文档与真实实现一致。

3. **做更多真实项目验证**  
   证明这套分层和 workflow 不只对 Prism 自身成立。

---

## 结论

如果用一句偏克制但明确的话来概括：

> Prism 1.0 已经是一个成立的、很有极客味道的个人 AI 工作流管线；当前主要欠账在叙事补齐与进一步泛化验证，而不在核心架构本身。
